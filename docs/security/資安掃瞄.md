- 安全目標：找出可被利用的程式碼弱點、危險依賴、檔案處理風險、PDF 惡意檔案風險、AI/OCR/agent 相關風險、Windows 打包與子行程風險

請依照以下流程審查：

1. 建立攻擊面清單
請先閱讀 README、CLAUDE.md、docs/ARCHITECTURE.md、requirements.txt、optional-requirements.txt、main.py，以及 controller/、model/、src/、utils/、view/、scripts/ 目錄。列出所有安全敏感入口點：
- 使用者可控制的檔案路徑、檔名、拖放輸入

2. 深度檢查高風險弱點
請逐檔檢查下列類型，並引用具體檔案與行號：
- Path traversal：使用者可否透過檔名、匯出路徑、暫存路徑覆寫任意檔案
- Unsafe temp file：暫存檔/目錄是否可預測、競態條件、未安全刪除
- Command injection：subprocess 是否使用 shell=True 或拼接命令
- Arbitrary file read/write/delete：開啟、儲存、匯出、刪除、覆寫流程是否缺少限制
- Malicious PDF handling：PDF parser 是否可能因惡意 PDF 造成 crash、DoS、記憶體暴漲、超大頁面/圖片炸彈
- Image/OCR risks：Pillow、OCR、torch、模型載入是否處理不可信輸入；是否可能大量耗用 GPU/CPU/RAM
- Dependency vulnerabilities：requirements.txt / optional-requirements.txt 是否 pin 版本；是否存在已知 CVE 或供應鏈風險
- Insecure deserialization：pickle、marshal、shelve、joblib、yaml.load 等
- Unsafe dynamic import / plugin loading
- Secrets leakage：repo 中是否有 API key、token、私鑰、憑證、路徑、使用者個資
- Logging/privacy：是否把檔案內容、OCR 文字、路徑、例外堆疊寫入 log
- LLM/agent prompt injection：若專案使用 GPT agents，檢查 PDF 內容或 OCR 文字是否可能被當作指令，導致工具濫用、資料外洩或任意檔案操作

6. 請額外輸出
- Top 10 優先修補清單
