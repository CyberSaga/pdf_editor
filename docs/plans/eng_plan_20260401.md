# gstack Plan Eng Review - Week 1 Implementation

工程審查著重於技術實作的嚴謹性、邊界案例 (Edge Cases) 以及系統穩定性。

## 技術發現與修正 (Architecture & Implementation)

### 1. 縮放信號缺失 (The Zoom Gap)
*   **發現**：原計畫僅掛載於 `sig_viewport_changed`（捲動信號）。但縮放 (Zoom) 會觸發 `_rebuild_continuous_scene_scaled`，該路徑會停用捲動信號。
*   **修正**：必須額外連結 `sig_scale_changed` 信號。確保使用者縮放後，框線位置與大小能正確重新繪製，不會產生偏移 (Drift)。

### 2. 游標狀態衝突 (Cursor Collision)
*   **發現**：在拖曳文字塊時，IBeam 游標可能會覆蓋 `ClosedHandCursor`。
*   **修正**：在設定 IBeam 游標前，必須檢查 `_text_edit_drag_state == IDLE`。

### 3. 資料結構優化
*   **調整**：將 `_block_outline_items` 從單純的 `list` 改為 `dict[(page_idx, block_idx)]`。
*   **理由**：為了實作「滑鼠懸停時隱藏虛線框（改顯示實線亮顯）」的精確互動，需要 O(1) 的查詢能力。

## 強制測試覆蓋 (Mandatory Tests)
Eng Review 強制要求在 `test_text_editing_gui_regressions.py` 中加入以下測試：
1.  `test_mode_switch_commits_edit_not_discards`：驗證切換模式時自動儲存。
2.  `test_escape_still_discards`：驗證 ESC 鍵仍維持顯式「放棄」行為。
3.  `test_block_outlines_only_drawn_for_visible_pages`：驗證效能保護機制，確保不處理非可視頁面。
4.  `test_cmd_shift_z_fires_redo`：驗證 macOS 標準重做快捷鍵。

## 互動要求 (UI/UX Requirement)
*   **Toast 提示**：模式切換自動儲存時，必須顯示 1.5 秒的「Text edit saved.」Toast，避免使用者對於「文字為何突然固定」感到困惑。

## 最終 verdict
**READY** - 補齊了縮放處理與回歸測試，架構足以支撐複雜 PDF 操作。
