## Codex 分析

• the key was treating visual stability as a hard invariant, not a side effect.

  What made it work:

  1\. Freeze glyph anchors during editing  
     Keep each glyph’s baseline/advance anchored in a stable coordinate system, so edits don’t reflow unrelated characters.  
  2\. Deterministic text shaping  
     Same input \+ same font state always yields the same glyph positions (no hidden fallback/path differences between  
     frames).  
  3\. Incremental layout only  
     Recompute only the edited span; untouched runs keep their original cached metrics and transforms.  
  4\. Rounding discipline  
     Avoid repeated float↔int conversions that accumulate drift and create tiny “jumps.”  
  5\. Single source of truth for metrics  
     One canonical place for font size, kerning, line height, transform matrix — no duplicate calculations in render vs edit  
     paths.  
  6\. Guardrails in tests  
     Snapshot/position-delta checks for cursor moves, insert/delete, IME composition, zoom changes, and undo/redo to catch  
     regressions fast.

  Mindset while writing this version:

  \- “No movement unless text semantics changed.”  
  \- “Every frame should be idempotent.”  
  \- “If it moved, we can point to exactly why.”

  1\. Overview — Five Layers of Fixes

  The "glyph jumps on each edit" symptom turned out to be a stack of failures across the view, model, and the preview/commit  
   boundary. The 47 commits in this range collectively address it on five layers. Each layer is independently necessary;  
  removing any one would put a visible jump back.

  Layer A — Commit-side fidelity (model/pdf\_model.py)

  \- Float font sizes end-to-end. EditTextRequest / MoveTextRequest / TextEditSession.current\_size all carry float. Helpers  
  \_parse\_font\_size\_str / \_format\_font\_size keep fractional sizes through the combo widget. (c091661f)  
  \- Faithful line height. \_apply\_redact\_insert derives \_line\_ht from member\_spans — median baseline advance for multi-line,  
  max bbox height for single-line — and threads it into \_build\_insert\_css as an explicit line\_height pt. (fbfe4c6)  
  \- Honor tight leading. \_build\_insert\_css previously did line\_height \= round(max(size, line\_height), 2\) unconditionally,  
  silently raising explicit tight values (8pt advance on a 10pt font) to the font size. Now the max(size, …) floor only  
  applies on the auto-calculate branch (line\_height \<= 0). (c1d8d79)  
  \- Don't widen the wrap. get\_render\_width\_for\_edit returns float(rect.width) directly. Qt's slightly different horizontal  
  glyph metrics make any extra width re-break the lines. (1ffe189)  
  \- Trust the probe; subtract MuPDF overhead. Single-line edits were pushing surrounding text down by 20pt+ because the  
  pre-push probe height was clamped to a line\_count × size × 2 \+ size × 2 floor and MuPDF's fixed 2.0pt leading wasn't  
  accounted for. Both removed. (cc55872)  
  \- Shared path classification. \_classify\_insert\_path(...) is now a module-level function used by both the preview and  
  \_apply\_redact\_insert, so preview can never pick fast while commit picks htmlbox (or vice versa). (b97dd62 / b11d179)  
  \- Empty member\_spans → htmlbox. Prevents min() crash and a wrong fast-path classification. (ba6b8fb)

  Layer B — Real preview rasterization (PreviewRenderer.render)

  \- Replaces the prior blank-QImage stub with a real MuPDF render: opens a temp page sized rotation-aware to rect\_pt, builds  
   the CSS+HTML using the same model.\_build\_insert\_css / \_convert\_text\_to\_html the commit path uses, calls  
  insert\_htmlbox(..., rotate=normalized\_rotation, scale\_low=1), then get\_pixmap(matrix=fitz.Matrix(render\_scale,  
  render\_scale), alpha=True) and detaches via .copy() before closing the temp doc. (b97dd62)  
  \- Falls back to a minimal Helvetica CSS when model is None or lacks \_build\_insert\_css (test fakes).  
  \- Font-family aliasing for helv/helvetica to keep parity.

  Layer C — DPI-correct Qt font sizing

  \- \_display\_font\_pt(pdf\_pt, render\_scale) \= pdf\_pt × render\_scale × 72 / \_widget\_logical\_dpi() — equates MuPDF's 72 ×  
  render\_scale rasterization with Qt's point × logical\_dpi/72 widget pixels. Used for the editor QFont and the QTextDocument  
   height probe. PDF-pt remains the stored unit; only the display path is DPI-corrected. (da2dd8e)  
  \- \_measure\_text\_content\_height\_px uses display\_font\_pt. Measuring with the pdf-pt value under-reports.  
  \- editor.font \= qt\_font\_obj method-shadow removed — it was overwriting the QTextEdit.font() method with a QFont object and  
   crashing all subsequent font/size handler calls. (0845b4a)

  Layer D — Frozen-frame proxy geometry (the centerpiece)

  \- create\_text\_editor grabs the underlying viewport pixels at the editor's intended scene position and freezes them as  
  \_frozen\_first\_frame\_image before the proxy is shown. Result: the editor's first paint is a pixel-identical capture of the  
  PDF region — there is literally no source of jump on the open transition. (a975545)  
  \- For 90/270 rotations, the grab is the axis-aligned PDF bbox counter-rotated by QTransform().rotate(-90/+90) so the  
  post-setRotation widget lands back on the correct PDF pixels (the original blank-on-vertical bug from vertical-texts.pdf).  
  \- \_compute\_editor\_proxy\_layout no longer applies minimum-height clamps (MIN\_EDITOR\_HEIGHT\_PX). The first frame is forced  
  to track scaled\_rect.height exactly. (a975545)  
  \- Wrap width \= exact PDF bbox for edit\_existing intent: scaled\_width \= int(round(rect.width \* rs)) (not the render-width  
  helper which adds margin). (a975545)  
  \- Paragraph mode uses measured wrapped content height (\_measure\_text\_content\_height\_px) because the paragraph rect can be  
  oversized vs. actual ink. (6fdfb94)  
  \- PreviewBackedInlineTextEditor.configure\_render\_context only applies its rect-derived size when no explicit frame has  
  been set; this preserves the geometry chosen by create\_text\_editor. (f2e36c7)  
  \- paintEvent two-branch contract. While text \== initial\_text, paint the frozen MuPDF capture; on mutation, paint the CSS  
  preview. The \_text\_matches\_initial flag is cached on textChanged (not recomputed per paintEvent), and there's a fail-safe:  
   if a mutated preview comes back effectively transparent, paint the frozen capture \+ native QTextEdit text with  
  autoFillBackground=False. (54466f21)

  Layer E — Cross-edit anchor (the final piece, 54466f21)

  This is what made repeated reopens stop drifting:  
  \- New per-session caches: DocumentSession.run\_reopen\_anchors: dict\[str, fitz.Rect\] and run\_reopen\_anchor\_sizes: dict\[str,  
  float\], keyed "{page\_idx}::{span\_id}".  
  \- \_resolve\_edit\_text\_target writes the anchor on the first run-mode edit (using target\_span.bbox / target\_span.size).  
  \- \_apply\_redact\_insert uses the anchor as base\_layout for the htmlbox/fast-path insert, and the  
  \_calculate\_shrunk\_rect-derived new\_layout\_rect is overridden back to the anchor when one is set.  
  \- After block\_manager.rebuild\_page(...), the anchor is migrated onto whichever new run scores best by (text\_match\_penalty,  
   distance\_sq\_to\_anchor\_center) — so the anchor follows the rebuilt span even when its span\_id changes.  
  \- get\_text\_info\_at\_point consults the anchor cache first: if a click lands in an anchored region, the returned TextHit  
  carries the anchor's bbox and original size, not the freshly-extracted span's drifted measurements.

  Net effect: N edit/commit/reopen cycles on the same run produce N identical widget rects — the cumulative-shrink failure  
  mode reported by manual QA is gone.

  1\. Negative controls. test\_geometry\_negative\_control\_x\_offset (+2px injected → must be detected),

  Real coverage gaps

  1\. Paragraph-mode reopen is uncovered. The reopen-cycle test always picks the first run-mode span. The anchor system that

  2\. Only 5 reopen cycles by default. REOPEN\_SESSION\_CYCLES=5 with REOPEN\_GEOMETRY\_DRIFT\_MAX\_PX=1. A 0.1 px/cycle drift

  3\. Save → close → reopen-file → edit is not covered. The anchor cache lives on DocumentSession and dies on close. The
