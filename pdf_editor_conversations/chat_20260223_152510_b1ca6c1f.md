## Step-by-step 根因分析

### Bug 1：縮放後文字模糊

**問題位置**：`_wheel_event`（第 673–681 行）

```python
def _wheel_event(self, event):
    if event.modifiers() & Qt.ControlModifier:
        factor = 1.1 if event.angleDelta().y() > 0 else 0.9
        self.scale *= factor
        self.graphics_view.setTransform(self.graphics_view.transform().scale(factor, factor))
        event.accept()
```

根本原因：觸控板縮放只對 `QGraphicsView` 套用 **view transform**（視圖矩陣縮放），讓現有 bitmap pixmap 被拉伸/縮小，**從未以新解析度重新渲染 PDF 頁面**。Qt 僅做雙線性插值 → 模糊。

---

### Bug 2：編輯模式點擊位置偏移

**問題位置**：`_scene_pos_to_page_and_doc_point`、`_update_hover_highlight`、`_create_text_editor` 等

```python
doc_y = (scene_pos.y() - y0) / self.scale   # ← BUG
```

**座標關係推導**：
- pixmap 以 `render_scale`（渲染時的 scale）產生 → 場景座標 = `PDF_點 × render_scale`
- View transform = 觸控板縮放後的累積倍數
- `mapToScene` 已補償 view transform → `scene_x = PDF_x × render_scale`
- **但程式碼除以 `self.scale`（view transform 累積值），而非 `render_scale`**

**具體反例**（render_scale=1.0，縮放到 self.scale=1.5）：
```
scene_x = mapToScene(150) = 150 / 1.5 = 100   ← 已是 PDF 坐標（因 render_scale=1.0）
doc_x   = 100 / self.scale = 100 / 1.5 = 66.7  ← 錯誤！
正確應為: 100 / render_scale = 100 / 1.0 = 100  ✓
```

---

## 修正方式選項

| 方案 | 說明 | 優點 | 缺點 |
|---|---|---|---|
| **A** | 每次 wheel 事件立即重渲所有頁面 | 最簡單 | 每次縮放都重渲，使用者拖動時卡頓 |
| **B** | 追蹤 `_render_scale`，縮放後 debounce 重渲 | 滑順+正確 | 需加 timer + 新 signal |
| **C** | 僅重設 view transform，永遠以 render_scale=1.0 渲染 | 簡單 | 大縮放時仍模糊（pixmap 無法放大清晰） |

**最佳選擇：方案 B**
- 觸控板縮放期間：view transform 即時更新 → 流暢視覺預覽
- 停止縮放後 300ms：重新以 `self.scale` 渲染清晰 PDF pixmap，view transform 重設為 1.0
- 座標轉換全程使用 `_render_scale`（實際 pixmap 渲染比例）→ 位置永遠正確
