## 修改摘要

### 行為變更

**之前**：在 `edit_text` 模式點擊文字 → 立即開啟編輯框 → 已有編輯框時才能拖曳

**之後**：點擊文字（無論有無現有編輯框）→ 進入「待定」狀態 → 放開即開啟編輯框 / 拖曳即移動

### 具體邏輯（統一的判定流程）

| 動作 | 有現有編輯框 | 無編輯框（點新文字塊）|
|------|------------|---------------------|
| **按下** | `_drag_pending=True`，記錄按下位置 | `_drag_pending=True`，存入 `_pending_text_info` |
| **放開（無移動）** | 在編輯框內定位游標 | 呼叫 `_create_text_editor` 開啟編輯框 |
| **拖曳（>5px）** | 進入 `_drag_active`，移動現有框 | 先建立編輯框，立即套用偏移量進入 `_drag_active` |

### 新增 / 修改的方法

1. **`__init__`**：新增 `_pending_text_info = None` 狀態變數
2. **`_mouse_press`**：無論是否已有編輯框，點擊文字塊都先設 `_drag_pending`，不立即開啟編輯框
3. **`_mouse_move`**：pending → drag 轉換時，若無編輯框則先建立再套用偏移量；所有移動均呼叫 `_clamp_editor_pos_to_page` 限制邊界
4. **`_mouse_release`**：`_drag_pending` 且無編輯框時 → 開啟編輯框；其餘邏輯不變
5. **`_clamp_editor_pos_to_page`**（新增）：將編輯框位置限制在目前頁面的場景座標範圍內，不可越界到其他頁面
6. **`set_mode`**：切換模式時同步清除 `_pending_text_info` 及所有拖曳狀態
7. **`_finalize_text_edit`**：重置時也清除 `_pending_text_info`
