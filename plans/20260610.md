## Open -- Audit remediation sequence (from 2026-06-10 two-round audit: codebase audit + defensive-programming review)

Findings from both rounds deduped into one ordered plan. Ordering rationale: restore the
test gate first (a red suite undermines every later verification), then the user-visible
breakage, then chokepoint guards (one fix bounds many call sites — per the established
"audit all sites + chokepoint + guard FIRST" rule), then memory budgets, then UI-thread
responsiveness, then hygiene. Each phase is independently shippable; within a phase items
share a pattern and should land together.

### Phase 0 — Restore trust in the gate

- [ ] **0.1 Bisect and fix the order-dependent test polluter.** Full `pytest test_scripts`
  run fails 7 tests in `test_no_jump_editor_geometry.py` (42.91% pixel diff vs 1%
  threshold); same file passes in isolation (377 passed). An earlier test leaks global
  state that corrupts render fidelity — prime suspect: the module-import-time
  `fitz.Page.get_text` rawdict monkey-wrap (`model/pdf_model.py:113`) or a font/DPR
  global. Bisect with `pytest --stepwise` / half-splitting, then scope the leaked state
  (fixture-scoped patch or reset). Everything below assumes this gate is green.

### Phase 1 — User-visible breakage (the optimize error dialog)

- [ ] **1.1 Make linearize capability-gated; delete the dead `linear=` fallback.**
  `save_optimized_working_doc` falls back to `fitz` save with `linear=1` when pikepdf is
  absent — an API removed in PyMuPDF 1.24+, so it always raises
  "Linearisation is no longer supported" (the screenshot bug). Expose an
  `optimize_capabilities()` query from the model; the dialog disables the
  linearize/object-streams options when pikepdf is missing; fail fast with a clear domain
  error before any work if options demand packaging that isn't available. Remove the
  `linear=` kwarg from all fitz saves. Env action: install pikepdf into `.venv` and add
  it to the dependency spec.
- [ ] **1.2 One error-wrapping layer (rides with 1.1, same code path).** Model raises
  domain exceptions with bare messages (e.g. `PdfOptimizeError`); only the controller adds
  the user-facing 「…失敗」 framing — fixes the doubled 「最佳化 PDF 失敗: 最佳化 PDF
  失敗:」 prefix. Also switch eager f-string logging to lazy `%s` in the touched handlers.

### Phase 2 — Chokepoint guards (OOM / logic-bypass closure)

- [ ] **2.1 Central render-scale clamp** — already tracked with a strict-xfail red-light
  test in place; see "F1 follow-up — central render-scale clamp gap" below. This is the
  decompression-bomb guard bypass on the main view path; land it first in this phase.
- [ ] **2.2 One shared zoom-limit constant in the view layer.** Wheel zoom is unbounded
  while pinch caps at 4.0 and the combo at 400% — three policies for one value. Define a
  single MAX/MIN zoom constant and apply it to all three input paths.
- [ ] **2.3 Foreign-document open guard.** Extract open-time checks (size cap, page cap,
  encryption handling) into a `_guard_foreign_doc(path)` helper; route `open_pdf`,
  `insert_pages_from_file`, and `headless_merge` through it. Add a post-merge invariant:
  resulting page count stays under `_MAX_PAGES`. Insert contiguous source runs with one
  `insert_pdf` call instead of per-page.
- [ ] **2.4 Page-number validation helper for every tool method.** `add_highlight`/
  `add_rect` accept page 0/negative and silently mutate the wrong page via Python
  negative indexing. One `_require_page(page_num)` helper used by all public tool
  methods; raise on out-of-range.
- [ ] **2.5 Watermark sanitization: one coercion chokepoint.** Harden `_coerce_wm`
  (finite-check all numerics — json accepts Infinity/NaN; validate color as exactly 3
  floats in [0,1]; bound offsets) and funnel `add_watermark`/`update_watermark` through
  the same coercion so all three entry points share one policy.
- [ ] **2.6 Single-instance forwarded-argv filter: validate relative tokens.** The
  acceptance filter only vets absolute paths; relative tokens pass unchecked to the
  receiving instance's open path. Resolve-then-validate every token against the
  .pdf-only contract.

### Phase 3 — Memory budgets (bytes, not counts)

- [ ] **3.1 Undo snapshot byte budget + dedup.** `SnapshotCommand` stores two full doc
  serializations per structural op; the stack caps count (100) not bytes → multi-GB
  growth on large docs. Dedup adjacent snapshots (command N's after == N+1's before),
  enforce a cumulative byte budget with oldest-first eviction; consider temp-file spill
  later.
- [ ] **3.2 Print snapshot to temp file, not BytesIO.** `build_print_snapshot` clones the
  whole doc in memory (~3× peak) and the dispatcher writes those bytes to a temp file
  anyway — pass a path end-to-end and drop the in-memory copy.

### Phase 4 — UI-thread responsiveness (reuse the existing worker pattern)

- [ ] **4.1 Thumbnails: all call sites through the batched async path.** The synchronous
  all-pages `_update_thumbnails` is called from ~9 sites (incl. every undo/redo) while
  `_schedule_thumbnail_batch` exists but is only used on open. Route all sites through
  the batch scheduler; for structural ops invalidate only the affected page range.
- [ ] **4.2 Search on a worker thread.** Full-doc search runs synchronously on the UI
  thread, O(pages × hits) native calls. Mirror the optimize worker/bridge pattern with
  incremental per-page hit signals and cancellation on new query.
- [ ] **4.3 (Low) Cache the overlay render page.** The watermark overlay path rebuilds a
  temp doc per repaint; cache keyed by (page, watermark revision) to stop per-frame
  copies during interactive zoom.

### Phase 5 — Hygiene / documentation

- [ ] **5.1 pyproject.toml reconciliation** — already tracked under "Repo governance";
  the audit re-confirmed the file is absent from the repo and git history while CLAUDE.md
  §3.1 treats it as the tooling source of truth.
- [ ] **5.2 Document cooperative OCR cancellation in PITFALLS.** Cancel is checked
  between pages only; a slow page cannot be interrupted — acceptable, but record it so
  it isn't mistaken for a hang.
