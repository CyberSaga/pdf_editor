  # Codex Adversarial Review

  Target: branch diff against 920368b
  Verdict: needs-attention

  No-ship: the new async search path creates a live-document concurrency hazard, and the controller only cancels it for a subset of mutations. That leaves real edit flows able to
  mutate the same `fitz.Document` the worker is reading, which can surface as crashes, corrupt reads, or stale search results under ordinary use.

  Findings:
  - [high] Async search can race with uncancelled document mutations (controller/pdf_controller.py:1879-2575)
    `search_text()` now spawns `_SearchWorker` against `self.model.doc`, and `_cancel_search()` is only called from a handful of entry points. Several real mutation flows in the
  same controller still do not stop an in-flight search first, including `add_highlight`, `add_rect`, `move_object`, `rotate_object`, `delete_object`, `resize_object`,
  `add_image_object`, `edit_text`, `move_text_across_pages`, and `add_annotation`. That means the worker can keep traversing pages while the main thread mutates the live PDF,
  which is exactly the read-during-mutation case the new comments warn about. The likely impact is intermittent PyMuPDF failures or hard-to-reproduce UI corruption when users
  search and edit at the same time.
    Recommendation: Make search cancellation a mandatory precondition for every document-mutating action, or move the guard into a single mutation chokepoint inside the
  model/command layer so no edit path can forget it.

  Next steps:
  - 1. Add a centralized mutation guard that cancels any in-flight search before document writes.
  - 2. Re-run the async search and edit-path tests under concurrent interaction to confirm the race is gone.

  This is review-only per the command, so I haven't changed anything. Worth noting: the finding is credible and maps exactly to your saved chokepoint rule — the Phase 4 implementation guarded the 8 mutation sites the plan enumerated plus 3 session-lifecycle boundaries, but per-site guarding missed the object/annotation/text-edit flows.
